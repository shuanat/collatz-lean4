## Ключевые концепции

### Централизованная архитектура

Проект использует **централизованную архитектуру** с тремя основными Core модулями:

- **`Foundations.Core`** - базовые математические определения
- **`Epochs.Core`** - определения эпох и связанных структур  
- **`SEDT.Core`** - константы и теоремы SEDT

### Система алиасов

Удобные сокращения для часто используемых определений в `Epochs.Aliases`:

```lean
-- Вместо длинных имен
Collatz.Foundations.depth_minus
Collatz.Epochs.TEpoch
Collatz.SEDT.α

-- Используйте короткие алиасы
Depth
TEpoch  
SlopeParam
```

### Иерархия зависимостей

```
Foundations.Core
    ↑
Epochs.Core ────┐
    ↑            │
SEDT.Core ───────┘
    ↑
Epochs.Aliases
    ↑
Специализированные модули
```

## Правила разработки

### ✅ Правильно

```lean
-- Импорт Core модулей
import Collatz.Foundations.Core
import Collatz.Epochs.Core
import Collatz.SEDT.Core
import Collatz.Epochs.Aliases

-- Использование алиасов
open Collatz.Epochs.Aliases (Depth StepType TEpoch)

-- Новые определения
def new_function (n : ℕ) : ℕ := sorry
```

### ❌ Неправильно

```lean
-- Прямой импорт специализированного модуля в Core
import Collatz.Epochs.Structure

-- Импорт между модулями одного уровня
import Collatz.Epochs.Structure
import Collatz.Epochs.TouchAnalysis

-- Дублирование определений
def depth_minus (n : ℕ) : ℕ := sorry  -- УЖЕ ЕСТЬ В Foundations.Core!
```
